# ADR-0002：原始事件存储与轮转策略

- 状态：Accepted
- 日期：2026-03-07
- 阶段：Phase 1.3

## 背景

采集链路在 1.2 阶段只保存在进程内队列，无法满足以下需求：
- 持久化审计（需要 append-only 原始轨迹）。
- 会话隔离（按日期与 session 管理）。
- 长时运行可控（需要轮转，避免单文件无限增长）。
- 异常中断恢复（重启后继续写入，不覆盖历史）。

## 决策

### 1) 存储介质与格式

- 存储路径根目录：`data/raw-events/`（可通过 CLI 参数覆盖）。
- 存储格式：`JSONL`，一行一个 `RawEvent`（`capture.raw.v0`）。
- 写入策略：append-only，每条事件序列化后追加换行。

### 2) 分区与命名

- 一级分区：本地时区日期目录 `yyyy-mm-dd`。
- 二级分区：session 文件分片。
- 命名规则：
  - 主分片：`{sessionId}.jsonl`
  - 轮转分片：`{sessionId}-r{NNNN}.jsonl`（从 `0001` 递增）

示例：
- `data/raw-events/2026-03-07/session-20260307-120001.jsonl`
- `data/raw-events/2026-03-07/session-20260307-120001-r0001.jsonl`

### 3) 轮转策略

- 同时支持大小和时间轮转：
  - 大小阈值：`--rotate-max-bytes`（默认 `1,048,576`）。
  - 时间阈值：`--rotate-max-seconds`（默认 `1,800`，`0` 表示禁用时间轮转）。
- 触发条件：任一条件命中即切换到下一分片。

### 4) 异常中断恢复

重启写入同一 `sessionId` 时：
- 扫描同日期目录下的该 session 分片。
- 若最新分片仍未触发大小/时间阈值，则继续追加。
- 若已触发阈值，则创建下一编号分片。

## 影响

- 满足 TODO 1.3 的 JSONL 落盘、session 分文件、轮转与续写要求。
- `jq`/脚本处理可直接消费分片文件（流式友好）。
- 后续可在 `scripts/validation` 追加 schema 校验，不影响现有文件组织。

## 备选方案（未采用）

1. 单文件无限追加：实现简单，但文件不可控增长，恢复与维护成本高。
2. SQLite 直存原始事件：查询友好，但不利于流式审计与外部脚本直接复用。
3. 仅按大小轮转：无法限制长期运行中“旧文件长期占用写入句柄”的问题。
